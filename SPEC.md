# SPEC.md — TextureCropOptimizer 完全仕様書

最終更新：初版
動作環境：Unity 2022.3.22f1（2022.3 LTS）

---

## 1. ツール概要

NDMFのOptimizingPhaseで動作するビルド時最適化ツール。
大きいテクスチャの中で実際に使用しているUV領域のみを切り出し、
テクセル密度を維持したまま必要最小の2のべき乗サイズに再マッピングする。

---

## 2. 全体方針

| 項目 | 仕様 |
|---|---|
| 元アセット | 完全無変更（ビルド時の一時変更は許容） |
| 処理タイミング | ビルド時のみ。AAO・TTT実行後のOptimizingPhase末尾 |
| 後処理 | ビルド終了後に複製物を破棄・一時変更を元に戻す |
| 品質方針 | 縮小による劣化なし・テクセル密度完全維持・未使用領域のみ削除 |
| 複雑化 | しない |

---

## 3. UV解析仕様

| 項目 | 仕様 |
|---|---|
| 対象チャンネル | UV0のみ |
| 有効範囲 | 0–1の範囲内のみ（±1e-4の浮動小数点誤差は許容してclamp）。範囲外の頂点が1つでもあればそのテクスチャをスキップ |
| 解析粒度 | UV島単位でAABBを算出し、全島のAABBの和集合をUsedRectとする |
| 複数メッシュ | 同一テクスチャを参照する全メッシュのUV0を和集合として扱う |

---

## 4. テクスチャ・マテリアル仕様

| 項目 | 仕様 |
|---|---|
| 対象シェーダー | VRChatデフォルト・Poiyomi・Liltoon |
| プロパティ取得 | `Material.GetTexturePropertyNames()` で動的取得 |
| UVチャンネル判定 | UV0以外が設定されているプロパティはスキップ |
| tiling/offset | デフォルト以外（tiling≠(1,1) または offset≠(0,0)）のプロパティのみスキップ。マテリアルは続行 |
| 非Readableテクスチャ | Graphics.Blit（GPU処理）のためRead/Write不要 |
| マテリアルコピー | 同一テクスチャを参照する全メッシュで1つのコピーを共有 |

---

## 5. 最適化閾値仕様

| 項目 | 仕様 |
|---|---|
| サイズ算出 | UsedRectの幅・高さそれぞれに対し、元テクスチャの対応する辺を基準に必要最小の2のべき乗サイズを独立算出する |
| 処理閾値 | 幅または高さの少なくとも一方が、元サイズより2のべき乗で1段階以上小さくなる場合のみ処理 |
| 非正方形対応 | 元テクスチャが非正方形（例: 256x1024）の場合、幅・高さを独立して最適化し非正方形の出力を生成する |
| 例 | 元4096x4096 → 幅2048, 高さ2048 → 処理する（両軸50%削減） |
| 例 | 元256x1024 → 幅256, 高さ512 → 処理する（高さのみ削減、AAO再パック後のケース） |
| 例 | 元4096x4096 → 幅4096, 高さ4096 → スキップ |

## 6. 設定コンポーネント仕様（TextureCropSettings）

### 配置

- Avatarルートに右クリックメニューから追加（`GameObject > TextureCropOptimizer > TextureCropSettings`）
- Avatarルートに1つのみ配置可能

### 検知ボタン

- インスペクター上に「テクスチャを検知」ボタンを表示
- 押下時にAvatar配下の全Rendererを走査
- UV解析を実行し2のべき乗で1段階以上削減できるマテリアルのみリストアップ
- 再スキャン時は同一マテリアルの既存設定を保持してマージ
- 効果がなくなったマテリアルはリストから削除
- 更新は手動（ボタンを押したときのみ）

### エントリUI（マテリアルごと）

```
[Material フィールド（ObjectField）]
[除外する チェックボックス]
[UV回転 ドロップダウン: 通常処理 / Emission固定]
```

| 状態 | 挙動 |
|---|---|
| 除外する = OFF | ドロップダウン有効 |
| 除外する = ON | ドロップダウンをグレーアウト・無効化。マテリアルを処理スキップ |

### UV回転ドロップダウン

| 値 | 意味 |
|---|---|
| 通常処理 | デフォルト |
| Emission固定 | UV島の回転・反転を禁止（将来のタイトパッキング機能追加時に有効化されるフラグ） |

## 7. 処理フロー

```
[NDMF OptimizingPhase / AAO・TTT実行後]
        ↓
1. TextureCropSettingsコンポーネントを取得
        ↓
2. Avatar配下の全RendererからMesh・Materialを収集
   └ 除外マテリアルをスキップ
        ↓
3. テクスチャ単位でグループ化
   └ 同一テクスチャを参照する全メッシュのUV0を収集
        ↓
4. UV島を検出 → 各島のAABBを算出 → 和集合でUsedRect確定
   └ 0–1範囲外の頂点が1つでもある → そのテクスチャをスキップ
        ↓
5. UsedRectから必要最小の2のべき乗サイズを算出
   └ 1段階以上小さくならない → スキップ
        ↓
6. テクスチャを複製 → UsedRect領域をGPU Blitで再構成
        ↓
7. メッシュを複製 → UV0をUsedRectに合わせてリマップ
        ↓
8. マテリアルを複製 → 全テクスチャプロパティを再構成テクスチャに差し替え
   └ tiling/offsetがデフォルト以外のプロパティはリマップ対象から除外
        ↓
9. ビルド終了後 → 複製物を破棄
```

## 8. エラー・ログ仕様

| 項目 | 仕様 |
|---|---|
| エラー時 | ビルドを中断 |
| ログ形式 | ツール起因と明示・Claude可読形式 |

### ログフォーマット

```
[TextureCropOptimizer] {LEVEL} | {Category} | {Message}
  Detail  : {詳細情報}
  Target  : {対象アセット名}
  Action  : {実行されたアクション or Build Aborted}
  Fix     : {修正方法のヒント（エラー時のみ）}
```

## 9. 将来実装予定（現在未実装）

以下はフラグ・設計上は考慮するが実装しない機能。

- **UV島タイトパッキング**：島の回転を使ったより高密度なパッキング。Emission固定フラグはこれ用。
- **プレビューウィンドウ・削減レポート**
- **UVヒートマップデバッグビュー**
- **チャンネルパックテクスチャのガード**
- **アニメーション（UVスクロール等）の検出と警告**
